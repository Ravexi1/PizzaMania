# Отчет о тестировании функционала PizzaMania

## Дата: 3 декабря 2025

## Исправленные проблемы

### 1. ✅ Переключение языка
**Проблема:** При переключении языка через форму в навбаре URL не менялся, всегда оставался `/ru/`

**Причина:** 
- Форма использовала `{% url 'set_language' %}`, который генерировал URL с учетом текущего языка
- Это приводило к тому, что форма всегда отправлялась на текущий языковой префикс

**Решение:**
- Изменили action формы на прямой путь `/i18n/setlang/`
- Этот путь находится вне `i18n_patterns`, поэтому работает корректно
- LocaleMiddleware уже был добавлен в настройках

**Файл:** `webapp/templates/webapp/base.html`

**Тесты:**
- ✅ Переключение на английский язык (en)
- ✅ Переключение на казахский язык (kk)
- ✅ Переключение на русский язык (ru)

### 2. ✅ Выбор размеров и добавок
**Проблема:** Нельзя было кликнуть на размер или добавку для их выбора

**Причина:**
- JavaScript обработчики были неправильно настроены
- Для размеров: использовался селектор `:has()` который не во всех браузерах работает одинаково
- Для добавок: клик по карточке программно переключал checkbox, но браузер мог обработать клик дважды

**Решение:**

**Размеры:**
```javascript
// Навешиваем обработчик change на radio кнопки
document.querySelectorAll('input[name="size"]').forEach(radio => {
    radio.addEventListener('change', function() {
        // Обновляем визуальное выделение
        // Пересчитываем цену
    });
});
```

**Добавки:**
```javascript
// Обработчик change на checkbox
checkbox.addEventListener('change', function() {
    // Обновляем визуальное состояние карточки
    updatePrice();
});

// Клик по карточке переключает checkbox
card.addEventListener('click', function(e) {
    if (e.target !== checkbox) {
        e.preventDefault();
        checkbox.checked = !checkbox.checked;
        checkbox.dispatchEvent(new Event('change'));
    }
});
```

**Файл:** `webapp/templates/webapp/product_detail.html`

**Тесты:**
- ✅ Клик по размеру выбирает его
- ✅ Клик по добавке переключает её
- ✅ Цена пересчитывается при выборе
- ✅ Визуальное выделение работает

### 3. ✅ Корзина с размерами и добавками
**Проблема:** Нужно было убедиться, что один товар с разными размерами/добавками хранится как отдельные позиции

**Решение:**
Логика уже была реализована в `views.py`:
- Создается уникальный ключ для каждой комбинации товара
- Ключ включает: product_id + size_id + addon_ids
- Товары с разными конфигурациями хранятся отдельно
- Товары с одинаковой конфигурацией увеличивают количество

**Пример ключа корзины:**
```python
cart_key = f"{product_id}_size_{size_id}_addons_{addon1_id}_{addon2_id}"
```

**Файлы:** 
- `webapp/views.py` (функция `add_to_cart`)
- `webapp/templates/webapp/cart.html` (отображение)

**Тесты:**
- ✅ Добавление товара с размером 30 см + 2 добавки
- ✅ Добавление того же товара с размером 25 см без добавок
- ✅ В корзине отображаются 2 отдельные позиции
- ✅ Каждая позиция имеет свою цену
- ✅ Повторное добавление той же конфигурации увеличивает количество
